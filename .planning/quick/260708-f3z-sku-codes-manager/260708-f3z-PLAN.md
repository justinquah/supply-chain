# Quick Task 260708-f3z: SKU Codes manager (variant→main mapping with conversion factor)

## Objective

Build a "SKU Codes" management UI so SCM/ADMIN can map alternate SKU codes (used in
inventory/sales files) to a MAIN product with a conversion factor stored in the
existing `sku_mappings` table. App code only — no schema/.sql/.planning changes.

## Context

- `sku_mappings(id, variant_sku, variant_name, main_product_id → products.id, units_per_variant numeric, notes, created_at, updated_at)` — already live.
- `units_per_variant` = main-SKU-equivalent units per ONE of the code; MAY be fractional.
- Importers (stock/actions.ts, sales/actions.ts) already resolve file SKU → main via `units_per_variant`.
- Auth pattern: `requireRole("SCM","ADMIN")` in page + every action; writes via `createAdminClient()`.
- Styling: Card/CardContent/CardHeader/CardTitle, Button, plain inputs/selects (see add-product-form.tsx).

## Tasks

### Task 1 — Server action module (type=auto)
- **files:** `src/app/(authed)/products/sku-codes/actions.ts`
- **action:** `"use server"`. Export `createSkuMapping`, `updateSkuMapping`, `deleteSkuMapping`.
  Each `await requireRole("SCM","ADMIN")` → `createAdminClient()`. Store `variant_sku` UPPERCASE+trim,
  `main_product_id`, `units_per_variant` (numeric, fractional allowed), optional `variant_name`/`notes`.
  Validate: variant_sku non-empty, main product chosen, factor > 0. On create, reject duplicate
  `variant_sku` case-insensitively with a clear error. `revalidatePath("/products/sku-codes")`.
- **verify:** `npm run build` compiles the module.
- **done:** three actions exist, gated, dedupe on create.

### Task 2 — Page + client components (type=auto)
- **files:** `src/app/(authed)/products/sku-codes/page.tsx`, `sku-codes-manager.tsx`
- **action:** Server page `requireRole("SCM","ADMIN")`; fetch all products (id, sku, name,
  product_family, is_active) and all `sku_mappings` joined to main product (`main_product_id → products(sku,name)`).
  Header "SKU Codes" + subtitle. Client component: Add form with main-product select, variant-code
  input, ratio input `[X] of this code = [Y] main unit(s)` (default 1/1) with live `→ factor Y/X`,
  optional variant name + notes. List grouped by main product with factor + plain-language ratio,
  inline Edit and Delete.
- **verify:** `npm run build` exits 0; route renders.
- **done:** page gated, add/list/edit/delete wired to actions.

### Task 3 — Entry point on /products (type=auto)

- **action:** Add "Manage SKU codes" link in header, visible to SCM/ADMIN only (`canManage`), → `/products/sku-codes`.
